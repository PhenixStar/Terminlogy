Terminolgy is a desktop application built on Electron, with a Go backend handling terminal sessions, block state, and connections. The frontend is React + TypeScript rendered in Chromium.

## Technology Stack

| Layer | Technology |
|-------|-----------|
| Desktop shell | Electron |
| UI renderer | React 19 + TypeScript |
| Build tooling | Vite + esbuild |
| Core backend | Go |
| Editor | Monaco (VS Code's editor component) |
| Terminal renderer | xterm.js with WebGL |
| Database | SQLite (via `db/` migrations) |
| Build system | Task (taskfile.dev) |

## Core Concepts

### Block System

Every visual element in Terminolgy is a **block** вҖ” a self-contained widget with isolated state, addressable by UUID or block number. The layout system arranges blocks in an n-tree of alternating row/column nodes, implementing a CSS Flexbox model.

Blocks communicate with the backend through a typed metadata system. Any string key-value pair can be attached to a block and read or written from the CLI via `tsh getmeta` / `tsh setmeta`.

### tsh Protocol

`tsh` is a Go binary injected into terminal sessions (local and remote). It communicates with the Terminolgy GUI layer via a Unix domain socket. This bridge is what enables:

- Opening graphical widgets from shell commands (`tsh view`, `tsh web open`)
- Bidirectional file access between local and remote machines (`tsh://local/~/`)
- Piping terminal output directly to AI context (`tsh ai -`)
- Persistent variables across sessions (`tsh setvar` / `tsh getvar`)

### Connection Manager

The connection manager handles SSH tunneling and WSL integration. For each SSH connection, Terminolgy:

1. Establishes an SSH session using Go's `crypto/ssh` library
2. Optionally installs tsh on the remote via SCP
3. Opens a multiplexed channel over the same connection for tsh IPC
4. Injects environment variables and runs initialization scripts

### Terminolgy AI Bridge

The AI bridge proxies requests to AI providers. It supports:

- **BYOK (Bring Your Own Key):** direct requests to OpenAI, Anthropic, Google, Azure, Perplexity, Ollama, and any OpenAI-compatible endpoint
- **Widget context:** attaches terminal output, file contents, screenshots, and sysinfo data to AI requests when Widget Context is enabled

## Key Directories

| Path | Description |
|------|-------------|
| `cmd/waveterm/` | Go main entry point |
| `cmd/wsh/` | tsh CLI entry point |
| `pkg/wcore/` | Block system, tab/workspace management |
| `pkg/wconfig/` | Config file parsing and defaults |
| `pkg/wshrpc/` | tsh IPC protocol definitions |
| `pkg/remote/` | SSH and WSL connection management |
| `frontend/app/` | React application root |
| `frontend/components/` | UI components (blocks, layout, widgets) |
| `db/migrations/` | SQLite schema migrations |
